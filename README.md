# Фотоконкурс Bitrix24 — руководство по развёртыванию

## Что это

Веб-приложение для еженедельного анонимного фотоконкурса внутри Bitrix24.
Развёртывается на Cloudflare Pages (бесплатно), встраивается в Bitrix24 через левое меню.

**Безопасность:** вебхук Bitrix24 хранится только в Cloudflare Secrets. В коде фронтенда его нет. Злоумышленник, скачав HTML, не получит доступ к API портала.

---

## Шаг 1. Подготовка Bitrix24

### 1.1. Создать смарт-процесс с нужными стадиями

Стадии (в точности как обсуждали):
- **Стадия 1** — новые элементы + основная карточка стартует здесь
- **Стадия 2** — голосование активно (для основной карточки)
- **Стадия 3** — участники текущего конкурса
- **Стадия 4** — закрытие голосования (для основной карточки)
- **Победная стадия** — элементы-победители
- **Проигрышная стадия** — элементы не-победители

### 1.2. Добавить поля на элементы смарт-процесса

| Поле | Тип |
|------|-----|
| Фото | Файл |
| Подпись к фото | Строка |
| Номер участника | Целое число |
| Имя участника | Строка |
| Количество голосов | Целое число |
| Победитель | Список (значения: Да / Нет) |
| Номер конкурса | Целое число |
| Дата конкурса | Дата |

**Важно:** значения списка "Победитель" должны называться именно `Y` и `N` (или `Да` и `Нет` — тогда в `wrangler.toml` нужно будет подставить реальные коды из Bitrix24).

### 1.3. Создать CRM-форму

Форма для заполнения сотрудниками. Поля: Фото, Подпись, Имя. Остальное (Номер участника, Номер конкурса, Дата конкурса) заполняется BP при создании.

### 1.4. Настроить бизнес-процессы

- **BP 1** — при создании элемента на стадии 1: заполнить Номер участника, Номер конкурса, Дата конкурса (из глобальных переменных, +1 к номеру, текущая дата)
- **BP 2** — по расписанию (например, воскресенье 19:50): переместить все элементы стадии 1 в стадию 3
- **BP 3** — по расписанию (например, воскресенье 20:00): переместить основную карточку на стадию 2, отправить в чат сообщение со ссылкой на приложение
- **BP 4** — по расписанию (например, вторник 20:00, через 48 часов): переместить основную карточку на стадию 4
- **BP 5** — триггер на переход основной карточки в стадию 4: для каждого элемента в стадии 3 проверить поле Победитель — если Да → переместить в Победную стадию, если Нет → в Проигрышную
- **BP 6** — после обработки: переместить основную карточку обратно на стадию 1 (новый цикл)

### 1.5. Создать входящий вебхук

1. Открыть: **Разработчикам → Другое → Входящий вебхук**
2. Назначить права: `crm` (полный доступ)
3. Скопировать полный URL вида:
   `https://ВАШ_ПОРТАЛ.bitrix24.ru/rest/1/abcdef123456xyz/`
4. Сохранить — понадобится для Cloudflare.

### 1.6. Записать все ID

Открой смарт-процесс, зайди в настройки и запиши:
- Entity Type ID смарт-процесса (цифра в URL, например `1032`)
- ID основной карточки (цифра, например `1`)
- ID каждой стадии — можно посмотреть через API-браузер Bitrix24 или в URL при редактировании стадии. Формат: `DT1032_10:UC_xxxxxx` или числовой
- API-имена всех полей (формат: `ufCrm_1_1234567890`) — видны в настройках полей смарт-процесса

## Шаг 2. Подготовка компьютера

### Что нужно установить

Для запуска этого проекта нужны два инструмента:

1. **Node.js** — это среда выполнения JavaScript. Без неё ничего не работает.
2. **wrangler** — это официальная командная строка Cloudflare. Через неё мы будем загружать код на хостинг и управлять базой данных.

### 2.1. Установить Node.js

1. Зайти на [nodejs.org](https://nodejs.org/)
2. Скачать версию с надписью **LTS** (это стабильная версия, рекомендуется)
3. Запустить установщик, везде нажать «Далее»
4. После установки открыть терминал и проверить:

```bash
node --version
```

Должна появиться строка вида `v20.11.0` (число должно быть 18 или выше).

> **Где открыть терминал?**
> - Windows: нажать `Win + R`, ввести `cmd`, нажать Enter — или найти «Командная строка» / «PowerShell» в меню Пуск
> - Или нажать правой кнопкой мыши на папку проекта → «Открыть в терминале»

### 2.2. Установить зависимости проекта

В терминале перейди в папку с проектом и установи wrangler:

```bash
cd d:\Claude\photocontest
npm install
```

Эта команда читает файл `package.json` и устанавливает всё необходимое в папку `node_modules`. Занимает 10–60 секунд. Если видишь сообщения про `npm warn` — это нормально, это просто предупреждения, не ошибки.

### 2.3. Создать аккаунт Cloudflare

Если ещё нет — зарегистрироваться бесплатно на [cloudflare.com](https://cloudflare.com). Бесплатный план полностью подходит для этого проекта.

### 2.4. Войти в Cloudflare через терминал

```bash
npx wrangler login
```

Что произойдёт:
1. В терминале появится сообщение и откроется браузер
2. Браузер попросит войти в аккаунт Cloudflare или подтвердить доступ
3. После подтверждения вернись в терминал — там будет написано `Successfully logged in`

> Если браузер не открылся автоматически — скопируй ссылку из терминала и вставь в браузер вручную.

## Шаг 3. Создание базы данных

Приложение хранит голоса в базе данных **Cloudflare D1** — это бесплатная SQLite-база прямо на инфраструктуре Cloudflare. Нам нужно её создать и подготовить таблицы.

### 3.1. Создать базу данных

В терминале (в папке проекта) выполни:

```bash
npx wrangler d1 create photocontest
```

Wrangler создаст базу данных и выведет что-то похожее на это:

```
✅ Successfully created DB 'photocontest' in region WEUR

[[d1_databases]]
binding = "DB"
database_name = "photocontest"
database_id = "abc123ef-1234-5678-abcd-ef1234567890"
```

### 3.2. Скопировать database_id в конфиг

Из вывода выше нужно скопировать только значение `database_id` (длинная строка с дефисами).

Открой файл `wrangler.toml` в любом текстовом редакторе (Блокнот, VS Code и т.п.) и найди строку:

```toml
database_id = ""
```

Замени пустые кавычки на скопированный ID:

```toml
database_id = "abc123ef-1234-5678-abcd-ef1234567890"
```

> **Важно:** ID у тебя будет другой — скопируй именно из своего вывода команды, не из этой инструкции.

### 3.3. Создать таблицы в базе данных

База создана, но пустая. Нужно создать таблицы. Выполни обе команды:

```bash
# Создать таблицы в локальной (тестовой) базе
npm run db:init

# Создать таблицы в облачной (рабочей) базе на Cloudflare
npm run db:init:remote
```

Если всё прошло успешно — в терминале будет что-то вроде:
```
✅ Executed 2 statements
```

Что за таблицы создаются (файл `schema.sql`):
- `votes` — хранит каждый голос. Уникальное ограничение `UNIQUE(user_id, contest_id)` физически не даёт проголосовать дважды.
- `participant_counts` — хранит счётчик голосов по каждому участнику. Нужна для быстрого чтения и атомарного прибавления (+1 без гонки данных).

## Шаг 4. Заполнение конфига

Открой файл `wrangler.toml` — он находится в корне папки проекта. Заполни все пустые строки значениями из Bitrix24, которые записал на шаге 1.6.

```toml
[vars]
ENTITY_TYPE_ID = "1032"                       # ID смарт-процесса
MAIN_CARD_ID = "42"                           # ID основной карточки
STAGE_2_ID = "DT1032_10:UC_VOTING"           # стадия голосования
STAGE_3_ID = "DT1032_10:UC_PARTICIPANTS"     # стадия участников
STAGE_4_ID = "DT1032_10:UC_CLOSED"           # стадия закрытия
WINNER_STAGE_ID = "DT1032_10:SUCCESS"        # победная стадия
LOSER_STAGE_ID = "DT1032_10:FAIL"            # проигрышная стадия

FIELD_PHOTO = "ufCrm_1_1234567890"           # API-имя поля "Фото"
FIELD_CAPTION = "ufCrm_1_1234567891"         # API-имя поля "Подпись"
FIELD_PARTICIPANT_NUM = "ufCrm_1_1234567892" # API-имя поля "Номер участника"
FIELD_NAME = "ufCrm_1_1234567893"            # API-имя поля "Имя участника"
FIELD_VOTE_COUNT = "ufCrm_1_1234567894"      # API-имя поля "Количество голосов"
FIELD_IS_WINNER = "ufCrm_1_1234567895"       # API-имя поля "Победитель"
FIELD_CONTEST_NUM = "ufCrm_1_1234567896"     # API-имя поля "Номер конкурса"
FIELD_CONTEST_DATE = "ufCrm_1_1234567897"    # API-имя поля "Дата конкурса"
```

> **Где найти API-имена полей?**
> В Bitrix24: CRM → нужный смарт-процесс → Настройки → Поля → открыть любое поле → в строке «Код» или «Символьный код» будет что-то вроде `ufCrm_1_1715000000`. Это и есть API-имя.

> **Как узнать ID стадий?**
> Способ 1: Перейди в смарт-процесс → нажми на название стадии → посмотри URL в браузере.  
> Способ 2: Через встроенный API-браузер Bitrix24 (Разработчикам → API-браузер) вызови `crm.status.list` или посмотри в настройках воронки.

## Шаг 5. Деплой на Cloudflare Pages

### 5.1. Первый деплой

```bash
npx wrangler pages deploy public --project-name=photocontest
```

Эта команда загружает папку `public` (там лежит `index.html`) на Cloudflare Pages. При первом запуске wrangler создаст новый проект автоматически.

После завершения в терминале появится URL вида:
```
✨ Deployment complete! Take a peek over at https://photocontest.pages.dev
```

Сохрани этот URL — он понадобится для регистрации приложения в Bitrix24.

### 5.2. Записать секрет — URL вебхука

Это самый важный шаг для безопасности. Вебхук Bitrix24 нигде не должен быть в коде — только в зашифрованном хранилище Cloudflare.

```bash
npx wrangler pages secret put BITRIX_WEBHOOK_URL --project-name=photocontest
```

Wrangler спросит значение — вставь полный URL вебхука из шага 1.5 (с `/` на конце):

```
? Enter a secret value: › https://ВАШ_ПОРТАЛ.bitrix24.ru/rest/1/abcdef123456/
```

После этого ключ будет зашифрован и доступен только серверной части приложения.

### 5.3. Привязать базу данных и переменные к проекту

Cloudflare Pages нужно сообщить, что наш проект использует D1-базу и переменные из `wrangler.toml`. Это делается через веб-интерфейс:

1. Зайти на [dash.cloudflare.com](https://dash.cloudflare.com)
2. Слева: **Workers & Pages** → нажать на проект `photocontest`
3. Вкладка **Settings**

**Привязать базу данных:**
- Раздел **Bindings** → кнопка **Add** → выбрать **D1 database**
- Variable name: `DB`
- D1 database: выбрать `photocontest` из списка
- Нажать **Save**

**Добавить переменные окружения:**
- Раздел **Environment variables** → кнопка **Add** → **Production**
- Добавить все переменные из секции `[vars]` файла `wrangler.toml` (ENTITY_TYPE_ID, MAIN_CARD_ID, все STAGE_*, все FIELD_*)
- Для каждой: вставить название и значение → **Save**

> **Зачем это нужно?** `wrangler.toml` — это конфиг для локальной разработки. Для работы в облаке (production) переменные нужно указать отдельно в настройках проекта на Cloudflare.

### 5.4. Финальный деплой

После добавления всех настроек сделай ещё один деплой, чтобы всё применилось:

```bash
npx wrangler pages deploy public --project-name=photocontest
```

## Шаг 6. Регистрация приложения в Bitrix24

Чтобы приложение появилось в левом меню Bitrix24 и получало авторизацию пользователя, нужно зарегистрировать его как «локальное приложение»:

1. **Разработчикам → Другое → Локальное приложение**
2. Параметры:
   - **Использует только API** — нет
   - **Путь вашего обработчика**: `https://photocontest.pages.dev/`
   - **Путь для первоначальной установки**: `https://photocontest.pages.dev/`
   - **Обработчик**: `https://photocontest.pages.dev/`
   - **Права (scope)**: `crm`
   - **Меню слева**: Да (для placement `LEFT_MENU`)
3. Нажать **Сохранить** и затем **Установить приложение на портал**

После установки в левом меню Bitrix24 появится пункт с твоим приложением.

## Шаг 7. Настройка ссылки в чате

В BP 3 (который отправляет сообщение в чат при переходе в стадию 2) укажи ссылку на приложение внутри Bitrix24:

```
https://ВАШ_ПОРТАЛ.bitrix24.ru/marketplace/app/ID_ПРИЛОЖЕНИЯ/
```

ID приложения виден в URL после регистрации. Эта ссылка лучше прямой ссылки на Cloudflare — через неё BX24.js подхватывает авторизацию автоматически.

## Шаг 8. Тестирование

### 8.1. Боевой тест

1. Создать 2–3 тестовых элемента на стадии 3
2. Переместить основную карточку на стадию 2
3. Открыть приложение через левое меню Bitrix24
4. Проверить что участники отображаются и голосование работает
5. Попытаться проголосовать повторно — должна быть ошибка «Вы уже проголосовали»
6. Переместить основную карточку на стадию 4
7. Открыть приложение снова → должен отобразиться архив

### 8.2. Локальный запуск (для разработчика)

```bash
npm run dev
```

Откроется `http://localhost:8788`. Покажет «Приложение должно быть открыто внутри Bitrix24» — это нормально, в локальном режиме нет авторизации Bitrix24.

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| «Приложение должно быть открыто внутри Bitrix24» | Открывай через левое меню портала, не напрямую по URL |
| «Невалидный auth_id» | Токен BX24.js истёк, обнови страницу |
| «BITRIX_WEBHOOK_URL не задан» | Выполни `wrangler pages secret put BITRIX_WEBHOOK_URL` |
| Фото не загружаются | Проверь что API-имя `FIELD_PHOTO` правильное |
| «Уже проголосовали» но не голосовал | Проверь в D1: `SELECT * FROM votes WHERE user_id='...'` |
| Кнопка «Проголосовать» не работает | F12 → Network, смотри ошибку в /api/vote |
| Переменные не применяются | Добавь их в Cloudflare Dashboard (шаг 5.3) и сделай повторный деплой |
| `npm: command not found` | Node.js не установлен или не добавлен в PATH — переустанови с nodejs.org |
| `wrangler: command not found` | Запусти `npm install` в папке проекта |

## Технические детали

- **Лимиты Cloudflare (free):** 100k запросов/день к Functions, 25M чтений/день к D1 — достаточно с огромным запасом
- **Лимит Bitrix24 REST:** 2 запроса/сек на портал — используем `batch` для массовых обновлений
- **Один голос на пользователя** — гарантирован на уровне БД (`UNIQUE(user_id, contest_id)`)
- **Анонимность:** в Bitrix24 не пишется, кто за кого голосовал — только итоговые счётчики
- **Фото:** проксируются через бэкенд → URL вебхука не виден клиенту
